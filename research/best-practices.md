# 调研资料 — GitHub 个人品牌最佳实践

> 调研日期：2026-07-02
> 资料来源：多个英文/中文技术博客、开发者社区、GitHub 官方资源

---

## 一、核心发现

### 1.1 数据驱动的洞察

| 数据点 | 来源 | 含义 |
|--------|------|------|
| 89% 的开发者忽略 Profile README | GitHub 社区 | 用了就是 Top 11% |
| 73% 的招聘者会看 Pin 的仓库 | 行业调研 | Pin 策略至关重要 |
| 有技术博客 → 求职联络 3x，薪资高 25% | 行业调研 | 博文是杠杆 |
| 87% 的招聘者看重开源贡献 | Stack Overflow Survey | 开源是信号 |
| 一致贡献图 > 每天提交 | 多个来源 | 质量比频率重要 |
| Bio 是最高频曝光位 | GitHub UI | 第一优先优化 |

### 1.2 非码农的关键差异

- **纯开发者**：靠代码量/开源贡献/技术深度证明实力
- **复合型人才**（Will 的情况）：靠领域知识交叉 × 技术能力 × 方案展示
- 调研中**没有**针对"技术+业务复合型人才"的现成最佳实践，需要自己定义

## 二、五层曝光金字塔（定制版）

### 第1层：基础卫生
- 头像：专业半身照
- Name：中英双语
- Bio：黄金公式 = 角色定位 + 核心领域 + 价值主张
- Location / Blog / Social links

### 第2层：一句话定位
- 出现在 GitHub 搜索结果、用户卡片、Google 搜索
- 必须包含行业关键词（行为分析、可用性工程、AI Agent）

### 第3层：Profile README
- 同名仓库的 README.md
- 立体名片，30 秒传递核心信息
- 结构建议：
  1. 一句话定位
  2. What I Do（价值描述，非技能罗列）
  3. Expertise（领域知识，用图标/卡片）
  4. Featured Projects（带成果描述）
  5. Recent Writing / Talks
  6. Contact

### 第4层：精选项目仓库
- Pin 3-6 个仓库，讲一个连贯的能力故事
- 每个仓库必须有：高质量 README + 问题背景 + 解决方案 + 效果
- 仓库分类：技术方案 / 工具脚本 / 知识库 / 开源贡献

### 第5层：深度思想输出
- 技术博客，自建 + 社区平台同步
- 70-20-10 内容框架：教程70% / 经验20% / 观点10%
- 持续性 > 爆款

## 三、Profile README 设计原则

### 推荐做法 ✅
- **价值导向**：不说"我会 Python"，说"用 Python 构建了XX系统"
- **量化成果**：数字 > 描述
- **领域交叉**：强调稀缺的技能组合
- **中英双语**：覆盖国内+国际受众
- **简洁克制**：不用花哨动画/Snake游戏，专业顾问调性
- **GitHub Stats 卡片**：适量使用，展示技术活跃度
- **技能图标 Badge**：shields.io / skill-icons，视觉化技术栈

### 避免做法 ❌
- 纯技能罗列（没有上下文）
- 过多动画/GIF（分散注意力）
- 空洞的自我评价（"热爱学习""积极向上"）
- 只有英文或只有中文
- 没有联系方式

## 四、项目仓库策略

### Pin 策略框架
| 类型 | 数量 | 目的 |
|------|------|------|
| 深度展示 | 2 | 证明你能解决复杂问题 |
| 广度展示 | 2 | 证明你技术栈多样 |
| 近期创新 | 1 | 证明你与时俱进 |
| 社区贡献 | 1 | 证明你参与生态 |

### 仓库 README 最佳结构
1. 项目名称 + 一句话说明
2. 问题背景（为什么做）
3. 解决方案（怎么做的）
4. 技术栈（用了什么）
5. 效果展示（截图/GIF/数据）
6. 安装/运行指南
7. License + 贡献指南

---

## 五、内容策略

### 70-20-10 框架
- **70% 教程/实战**：解决具体问题，可复用
- **20% 经验分享**：项目复盘、行业观察
- **10% 观点/评论**：趋势判断、立场表达

### 平台选择
- **主阵地**：GitHub Pages（自建博客）
- **分发渠道**：知乎、CSDN、LinkedIn、微信公众号
- **互动渠道**：Twitter/X、相关技术社群

### 频率建议
- 初期：每月 1-2 篇
- 稳定后：每周 1 篇
- 关键原则：持续性 > 数量

## 六、行动路线图

### 第一阶段（第1-2周）：基础建设
- [ ] 填写 GitHub Profile 基础信息（头像、Name、Bio、Location）
- [ ] 创建同名仓库 + Profile README
- [ ] Pin 现有优质项目

### 第二阶段（第1-2个月）：内容充实
- [ ] 精选项目补全 README
- [ ] 考虑开源 Agentic AI PoC 精简版
- [ ] GitHub Pages 站点搭建

### 第三阶段（第3个月起）：持续输出
- [ ] 发布第一篇技术博客
- [ ] 建立内容节奏
- [ ] 开始社区互动

### 第四阶段（第6个月+）：品牌深化
- [ ] 整合博客文章为电子书/专题
- [ ] 寻求演讲/分享机会
- [ ] 主动建立行业连接
